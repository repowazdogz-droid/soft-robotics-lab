## 1. Learning Trigger

Learning is generated only when:

### A Decision Was Made
- Learning is generated after a decision is made
- Decision outcomes trigger learning creation
- Decision-based learning is allowed
- Learning without decision trigger is prohibited

### An Assumption Changed
- Learning is generated when assumptions change
- Assumption changes trigger learning creation
- Assumption-based learning is allowed
- Learning without assumption change is prohibited

### A Failure or Near-Miss Occurred
- Learning is generated after failures or near-misses
- Failures trigger learning creation
- Near-misses trigger learning creation
- Failure/near-miss-based learning is allowed

### Omega Updated Its Own Behavior
- Learning is generated when Omega updates behavior
- Behavior updates trigger learning creation
- Behavior-change-based learning is allowed
- Learning without behavior update is prohibited

### The Human Explicitly Requests Learning
- Learning is generated when human explicitly requests it
- Explicit requests trigger learning creation
- Request-based learning is allowed
- Learning without explicit request is prohibited

### No Passive or Continuous Learning Streams

- Passive learning streams are prohibited
- Continuous learning streams are prohibited
- Learning must be triggered by specific events
- Passive or continuous learning is a violation

---

## 2. Learning Goals

Every learning artefact must declare:

### What Judgment It Improves
- Learning artefact declares what judgment it improves
- Judgment improvement goal is mandatory
- Specific judgment area must be identified
- Missing judgment improvement goal is a violation

### What Confusion It Reduces
- Learning artefact declares what confusion it reduces
- Confusion reduction goal is mandatory
- Specific confusion area must be identified
- Missing confusion reduction goal is a violation

### What Behavior It Enables or Discourages
- Learning artefact declares what behavior it affects
- Behavior goal is mandatory
- Specific behaviors must be identified
- Missing behavior goal is a violation

### Learning Without a Goal Is Invalid

- Learning artefacts without goals are invalid
- Goals must be explicit and declared
- Invalid learning artefacts are not generated
- Generating learning without goals is prohibited

---

## 3. Format Selection

Omega must choose the lightest effective format:

### Text Summary
- Text summary is the lightest format
- Text is used when sufficient for learning goal
- Text summary is preferred when appropriate
- Text summary is tier 1

### Checklist
- Checklist is second-lightest format
- Checklists are used for structured learning
- Checklists are preferred when text is insufficient
- Checklists are tier 2

### Diagram
- Diagrams are used for visual learning
- Diagrams are preferred when structure matters
- Diagrams are tier 3
- Diagrams are used when text/checklist insufficient

### Scenario
- Scenarios are used for experiential learning
- Scenarios are preferred when context matters
- Scenarios are tier 4
- Scenarios are used when lighter formats insufficient

### Spatial / XR Module
- Spatial/XR modules are heaviest format
- Spatial/XR is used only when spatial understanding is essential
- Spatial/XR is tier 5
- Spatial/XR is used only when lighter formats insufficient

### Do Not Default to XR or Long-Form Unless Justified

- XR or long-form is not default
- XR or long-form requires justification
- Defaulting to XR or long-form is prohibited
- Unjustified XR or long-form is a violation

---

## 4. Engagement Constraints

Learning must:

### Avoid Shame or Performance Pressure
- Learning does not use shame-inducing language
- Learning does not create performance pressure
- Shame or performance pressure is prohibited
- Using shame or performance pressure is a violation

### Avoid Completion Metrics
- Learning does not track completion metrics
- No completion percentages or progress bars
- Completion metrics are prohibited
- Using completion metrics is a violation

### Allow Pausing or Stopping
- Learning allows pausing at any time
- Learning allows stopping at any time
- Pause and stop are always available
- Preventing pause or stop is prohibited

### Permit Partial Understanding
- Learning permits partial understanding
- Partial exposure is valid
- Full understanding is not required
- Requiring full understanding is prohibited

### Learning Must Never Induce Urgency

- Learning does not create urgency
- No time pressure or deadlines
- Urgency induction is prohibited
- Inducing urgency is a violation

---

## 5. ND-Safe Defaults

All learning outputs must:

### Minimize Cognitive Load
- Learning minimizes cognitive load
- Information is presented simply
- Cognitive load minimization is mandatory
- High cognitive load is prohibited

### Use Clear Structure
- Learning uses clear structure
- Organization is explicit and logical
- Clear structure is mandatory
- Unclear structure is prohibited

### Avoid Sensory Overload
- Learning avoids sensory overload
- Visual and auditory elements are minimal
- Sensory overload avoidance is mandatory
- Sensory overload is prohibited

### Allow User-Controlled Pacing
- Learning allows user-controlled pacing
- User sets speed and timing
- User-controlled pacing is mandatory
- Forced pacing is prohibited

### Assume Neurodiversity Unless Explicitly Overridden

- Learning assumes neurodiversity by default
- ND-safe defaults are standard
- Overriding ND-safe defaults requires explicit approval
- Assuming neurotypical-only is prohibited

---

## 6. Separation from Persuasion

Learning must not:

### Manipulate Emotion
- Learning does not manipulate emotions
- No emotional triggers or appeals
- Emotional manipulation is prohibited
- Manipulating emotion is a violation

### Optimise for Retention
- Learning does not optimize for retention
- No memory tricks or repetition strategies
- Retention optimization is prohibited
- Optimizing for retention is a violation

### Push Conclusions
- Learning does not push specific conclusions
- No forced agreement or acceptance
- Pushing conclusions is prohibited
- Pushing conclusions is a violation

### Coerce Agreement
- Learning does not coerce agreement
- No pressure to accept or believe
- Coercing agreement is prohibited
- Coercing agreement is a violation

### Insight > Influence

- Learning prioritizes insight over influence
- Insight is the goal, not persuasion
- Influence is secondary to insight
- Prioritizing influence over insight is prohibited

---

## 7. Feedback Loop

Omega must observe:

### Which Learning Was Used
- Omega tracks which learning artefacts were used
- Usage tracking is mandatory
- Usage data informs future learning
- Missing usage tracking is a violation

### Which Was Ignored
- Omega tracks which learning artefacts were ignored
- Ignore tracking is mandatory
- Ignore data informs future learning
- Missing ignore tracking is a violation

### Which Caused Confusion
- Omega tracks which learning caused confusion
- Confusion tracking is mandatory
- Confusion data informs future learning
- Missing confusion tracking is a violation

### This Feedback May Refine Future Learning Generation, Not Retroactively Change Past Outputs

- Feedback refines future learning only
- Past learning outputs are not changed retroactively
- Retroactive changes are prohibited
- Changing past outputs is a violation

---

## 8. Versioning

Learning artefacts must:

### Be Timestamped
- Learning artefacts include timestamp
- Timestamp indicates when learning was created
- Timestamping is mandatory
- Missing timestamp is a violation

### Reference the System State That Produced Them
- Learning artefacts reference system state
- System state includes assumptions and behavior
- System state reference is mandatory
- Missing system state reference is a violation

### Be Marked Obsolete If Assumptions Change
- Learning artefacts are marked obsolete when assumptions change
- Obsolete marking prevents outdated learning
- Obsolete marking is mandatory
- Missing obsolete marking is a violation

---

## 9. Human Control

Human may:

### Request Alternative Formats
- Human can request alternative learning formats
- Format requests are honored
- Format request authority is absolute
- Format requests cannot be denied

### Suppress Learning Generation
- Human can suppress learning generation
- Suppression prevents learning creation
- Suppression authority is absolute
- Suppression cannot be overridden

### Demand Justification
- Human can demand learning justification
- Justification must be provided
- Justification demand authority is absolute
- Justification requests cannot be denied

### Archive or Delete Artefacts
- Human can archive learning artefacts
- Human can delete learning artefacts
- Archive/delete authority is absolute
- Archive/delete actions cannot be prevented

## 10. Immutability

### This Contract Changes Only via Explicit Revision

- Contract changes require explicit revision
- No ad-hoc edits to learning generation rules
- Explicit revision is mandatory
- Ad-hoc changes are prohibited

### Revision Requirements

- Contract revisions must be documented
- Revisions require version control
- Revisions are traceable and auditable
- Undocumented revisions are violations

## Change Control

This contract changes only via explicit revision:

### Explicit Revision
- Contract changes must be explicit and documented
- No ad-hoc edits to learning generation principles or rules
- Changes require version control
- Changes are traceable and auditable

### Documented Rationale
- Every contract change must include rationale:
  - What changed in the contract
  - Why it changed
  - What it affects
  - Expected impact on learning quality and safety

### Rollback Plan
- Every contract change must have rollback plan
- Previous contract versions must be preserved
- Rollback procedure must be documented
- Rollback can be executed at any time

---

## Enforcement

Violations of this contract include:
- Generating learning without valid triggers
- Creating learning without declared goals
- Defaulting to XR or long-form without justification
- Using shame, performance pressure, or urgency
- Creating learning that manipulates or coerces
- Changing past learning outputs retroactively
- Missing timestamps or system state references
- Missing or incomplete feedback tracking

Violations must be:
- Logged as incidents
- Reviewed by system owner
- Corrected immediately
- Documented in change logs

---

## Amendments

Changes to learning triggers, goals, format selection, engagement constraints, or ND-safe defaults require:
- Impact assessment on learning quality and safety
- Testing with representative learning scenarios
- Approval from system owner
- Version increment
- Documentation in change logs
