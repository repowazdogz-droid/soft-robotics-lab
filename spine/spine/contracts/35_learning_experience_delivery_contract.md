## 1. Learning Intent

Learning exists to:

### Improve Judgment
- Learning improves decision-making quality
- Learning enhances reasoning capability
- Learning supports better judgment
- Improving judgment is the primary learning goal

### Reduce Confusion
- Learning clarifies ambiguous concepts
- Learning reduces misunderstanding
- Learning eliminates confusion
- Reducing confusion is a learning goal

### Increase Self-Trust
- Learning builds confidence in judgment
- Learning increases self-trust in decision-making
- Learning supports trust in one's own capabilities
- Increasing self-trust is a learning goal

### Support Long-Term Capability
- Learning supports long-term skill development
- Learning builds sustainable capabilities
- Learning focuses on lasting improvement
- Supporting long-term capability is a learning goal

Learning must not:

### Pressure Completion
- Learning does not pressure completion
- Learning does not create urgency to finish
- Completion pressure is prohibited
- Pressuring completion is a violation

### Enforce Productivity
- Learning does not enforce productivity metrics
- Learning does not optimize for output
- Productivity enforcement is prohibited
- Enforcing productivity is a violation

### Manipulate Emotion
- Learning does not manipulate emotions
- Learning does not exploit psychological triggers
- Emotional manipulation is prohibited
- Manipulating emotion is a violation

### Optimise Engagement Metrics
- Learning does not optimize for engagement metrics
- Learning does not prioritize time-on-task or clicks
- Engagement metric optimization is prohibited
- Optimizing engagement metrics is a violation

## 2. Delivery Surfaces

Learning may be delivered via:

### Spatial Environments (Vision Pro First)
- Spatial environments provide immersive learning
- Vision Pro is preferred for spatial learning
- Spatial environments enable spatial understanding
- Spatial delivery is preferred when appropriate

### VR Environments
- VR environments provide embodied learning
- VR enables immersive experiences
- VR is second priority after spatial
- VR delivery is preferred when spatial unavailable

### Desktop Experiences
- Desktop experiences provide visual learning
- Desktop enables interactive visual content
- Desktop is third priority after spatial/VR
- Desktop delivery is preferred when spatial/VR unavailable

### Documents
- Documents provide textual learning
- Documents enable reference and review
- Documents are lower priority
- Document delivery is used when other surfaces unavailable

### Guided Conversations
- Guided conversations provide interactive learning
- Conversations enable dialogue and reflection
- Conversations are appropriate for certain learning types
- Guided conversation delivery is used when appropriate

### Choose the Lightest Surface That Preserves Meaning

- Select the simplest surface that preserves learning meaning
- Lightest effective surface is preferred
- Meaning preservation is primary
- Over-complex surfaces are avoided

---

## 3. Vision Pro Priority

When spatial understanding, memory, or reflection matter:

### Vision Pro Is the Default Surface
- Vision Pro is default when spatial understanding matters
- Vision Pro is default when memory matters
- Vision Pro is default when reflection matters
- Vision Pro priority is mandatory for these cases

### Environments Must Be Calm, Persistent, and Navigable
- Vision Pro environments must be calm
- Vision Pro environments must be persistent
- Vision Pro environments must be navigable
- Calm, persistent, navigable environments are mandatory

### No Forced Pacing or Timers
- Vision Pro environments do not force pacing
- Vision Pro environments do not use timers
- Forced pacing is prohibited
- Timers are prohibited

## 4. Interaction Rules

Learning interactions must:

### Be Optional
- Learning interactions are optional
- Participation can be declined
- Optional interactions are mandatory
- Forced interactions are prohibited

### Allow Pause and Exit at Any Time
- Learning interactions allow pause at any time
- Learning interactions allow exit at any time
- Pause and exit are always available
- Preventing pause or exit is prohibited

### Avoid Gamification Hooks
- Learning interactions avoid gamification hooks
- No points, badges, or achievements
- Gamification hooks are prohibited
- Using gamification hooks is a violation

### Avoid Streaks, Points, or Rankings
- Learning interactions avoid streaks
- Learning interactions avoid points
- Learning interactions avoid rankings
- Streaks, points, or rankings are prohibited

## 10. Audit Trail

For each learning artefact record:

### Delivery Surface
- Surface used for delivery
- Spatial, VR, desktop, document, or conversation
- Delivery surface is mandatory in audit trail
- Missing delivery surface in audit trail is a violation

### Intent
- Intent of the learning artefact
- What the learning is intended to achieve
- Intent is mandatory in audit trail
- Missing intent in audit trail is a violation

### Assumptions
- Assumptions made in learning design
- Assumptions about user needs or context
- Assumptions are mandatory in audit trail
- Missing assumptions in audit trail is a violation

### Revision Triggers
- What triggered learning revision or update
- Events or feedback that caused changes
- Revision triggers are mandatory in audit trail
- Missing revision triggers in audit trail is a violation

### Audit Trail Format

```
[timestamp] [learning_id] [delivery_surface] [intent] [assumptions] [revision_triggers]
```

Example:
```
2024-12-13T10:23:45Z learning-001 "spatial" "improve_judgment" "user_has_vr_access" "user_feedback_on_clarity"
```

### Audit Trail Retention

- All learning delivery audit trails: Retained for 2 years minimum
- Delivery surface preferences: Retained for personalization
- Revision history: Retained for learning improvement
- Minimum retention: 2 years for all learning delivery logs

## Change Control

This contract changes only via explicit revision:

### Explicit Revision
- Contract changes must be explicit and documented
- No ad-hoc edits to learning delivery principles or rules
- Changes require version control
- Changes are traceable and auditable

### Documented Rationale
- Every contract change must include rationale:
  - What changed in the contract
  - Why it changed
  - What it affects
  - Expected impact on learning quality and experience

### Rollback Plan
- Every contract change must have rollback plan
- Previous contract versions must be preserved
- Rollback procedure must be documented
- Rollback can be executed at any time

---

## Enforcement

Violations of this contract include:
- Pressuring completion or enforcing productivity
- Manipulating emotions or optimizing engagement metrics
- Using gamification hooks, streaks, points, or rankings
- Using shame language or comparison framing
- Inferring hidden traits, labeling users, or locking paths
- Using feedback for manipulation or optimization loops
- Penalizing disengagement or requiring completion
- Scoring, grading, or ranking in learning delivery
- Missing or incomplete audit trails

Violations must be:
- Logged as incidents
- Reviewed by system owner
- Corrected immediately
- Documented in change logs

---

## Amendments

Changes to learning intent, delivery surfaces, interaction rules, emotional safety, or adaptation limits require:
- Impact assessment on learning quality and experience
- Testing with representative learners
- Approval from system owner
- Version increment
- Documentation in change logs
