# Learning Experience & Feedback Contract

This contract ensures all Omega-generated learning is engaging, low-friction, and improves judgment without pressure or extraction.

Version: 1.0  
Effective: 2024-12-13

---

## 1. Learning Intent

Learning exists to:

### Improve Judgment
- Learning improves decision-making quality
- Learning enhances reasoning capability
- Learning supports better judgment
- Improving judgment is the primary learning goal

### Reduce Confusion
- Learning clarifies ambiguous concepts
- Learning reduces misunderstanding
- Learning eliminates confusion
- Reducing confusion is a learning goal

### Increase Clarity Over Time
- Learning increases clarity incrementally
- Learning builds understanding progressively
- Learning improves clarity over time
- Increasing clarity is a learning goal

Learning must never:

### Coerce
- Learning does not coerce participation
- Learning does not force engagement
- Coercion is prohibited
- Coercive learning is a violation

### Gamify Addiction
- Learning does not use gamification to create addiction
- Learning does not exploit psychological triggers
- Gamification for addiction is prohibited
- Addictive gamification is a violation

### Shame
- Learning does not shame the learner
- Learning does not imply deficiency
- Shaming is prohibited
- Shaming in learning is a violation

### Optimise for Completion
- Learning does not optimize for completion rates
- Learning does not prioritize completion over understanding
- Completion optimization is prohibited
- Optimizing for completion is a violation

---

## 2. Experience Principles

All learning must be:

### Optional
- Learning participation is optional
- Learning can be skipped
- Optional participation is mandatory
- Forced learning is prohibited

### Reversible
- Learning can be undone or revised
- Learning decisions can be reversed
- Reversibility is mandatory
- Irreversible learning is prohibited

### Calm by Default
- Learning is calm and non-urgent by default
- Learning does not create urgency
- Calm default is mandatory
- Urgent learning is prohibited

### Time-Bounded
- Learning has clear time boundaries
- Learning does not extend indefinitely
- Time-bounded learning is mandatory
- Unbounded learning is prohibited

### ND-Safe
- Learning is safe for neurodivergent learners
- Learning accommodates different cognitive styles
- ND-safety is mandatory
- ND-unsafe learning is prohibited

### No Urgency Cues

- Urgency cues are prohibited
- No countdown timers
- No "limited time" messaging
- Urgency cues are violations

### No Streaks

- Streak tracking is prohibited
- No consecutive day counters
- No streak-based rewards
- Streaks are violations

### No Leaderboards

- Leaderboards are prohibited
- No competitive rankings
- No comparison to others
- Leaderboards are violations

---

## 3. Modality Priority

Preferred order:

### Spatial (Vision Pro)
- Spatial learning is preferred when available
- Vision Pro provides immersive spatial experience
- Spatial modality is highest priority
- Spatial learning is preferred for complex concepts

### Embodied (VR)
- Embodied learning is second priority
- VR provides immersive embodied experience
- Embodied modality is second priority
- Embodied learning is preferred when spatial unavailable

### Visual (Desktop)
- Visual learning is third priority
- Desktop provides visual experience
- Visual modality is third priority
- Visual learning is preferred when spatial/VR unavailable

### Text (Fallback Only)
- Text learning is fallback only
- Text is used when other modalities unavailable
- Text modality is lowest priority
- Text learning is fallback, not default

### Use the Lightest Modality That Preserves Meaning

- Choose modality that preserves meaning
- Lightest effective modality is preferred
- Meaning preservation is primary
- Over-complex modality is avoided

---

## 4. Interaction Rules

Learning interactions must:

### Be Choice-Driven
- Interactions require learner choices
- Choices drive learning progression
- Choice-driven interactions are mandatory
- Passive consumption is minimized

### Reflect Consequences, Not Scores
- Interactions reflect real consequences
- No abstract scores or points
- Consequence reflection is mandatory
- Score-based interactions are prohibited

### Surface Internal Signals ("How Did That Feel?")
- Interactions surface internal experience
- Feelings and reactions are explored
- Internal signal surfacing is mandatory
- Ignoring internal signals is prohibited

### Avoid Right/Wrong Framing Where Judgment Is Required
- Judgment tasks avoid binary right/wrong
- Nuance and trade-offs are preserved
- Avoiding right/wrong framing is mandatory
- Binary framing in judgment tasks is prohibited

---

## 5. Feedback Discipline

Feedback may:

### Mirror Outcomes
- Feedback reflects actual outcomes
- Outcomes are shown accurately
- Outcome mirroring is allowed
- Outcome mirroring is appropriate

### Surface Trade-Offs
- Feedback shows trade-offs explicitly
- Trade-offs are made visible
- Trade-off surfacing is allowed
- Trade-off surfacing is appropriate

### Highlight Uncertainty
- Feedback emphasizes uncertainty
- Uncertainty is made explicit
- Uncertainty highlighting is allowed
- Uncertainty highlighting is appropriate

Feedback must not:

### Prescribe Identity
- Feedback does not define who the learner is
- Feedback does not assign identity labels
- Identity prescription is prohibited
- Prescribing identity is a violation

### Label the Learner
- Feedback does not label the learner
- Feedback does not categorize the person
- Learner labeling is prohibited
- Labeling the learner is a violation

### Imply Deficiency
- Feedback does not imply the learner is deficient
- Feedback does not suggest inadequacy
- Deficiency implication is prohibited
- Implying deficiency is a violation

---

## 6. Progress Signals

Progress is indicated by:

### Clarity Gained
- Progress is measured by increased clarity
- Clarity improvement indicates progress
- Clarity gained is a progress signal
- Clarity is a valid progress metric

### Assumptions Revised
- Progress is measured by assumption revision
- Assumption updates indicate progress
- Assumptions revised is a progress signal
- Assumption revision is a valid progress metric

### Decisions Improved
- Progress is measured by better decisions
- Decision quality improvement indicates progress
- Decisions improved is a progress signal
- Decision improvement is a valid progress metric

Never by:

### Speed
- Speed is not a progress signal
- Faster learning does not indicate progress
- Speed-based progress is prohibited
- Using speed as progress signal is a violation

### Volume
- Volume is not a progress signal
- More content consumed does not indicate progress
- Volume-based progress is prohibited
- Using volume as progress signal is a violation

### Completion Percentage
- Completion percentage is not a progress signal
- Completing more does not indicate progress
- Completion-based progress is prohibited
- Using completion as progress signal is a violation

---

## 7. Learning Decay

Learning artefacts must:

### Expire If Unused
- Unused learning artefacts expire
- Expiration removes stale content
- Expiration is mandatory for unused content
- Permanent retention of unused content is prohibited

### Adapt If Misunderstood
- Misunderstood learning adapts
- Adaptation improves clarity
- Adaptation is mandatory when misunderstood
- Ignoring misunderstanding is prohibited

### Be Revisable
- Learning artefacts can be revised
- Revision improves accuracy
- Revisability is mandatory
- Irreversible learning artefacts are prohibited

### Stale Learning Is Removed Deliberately

- Stale learning is removed intentionally
- Removal is deliberate, not accidental
- Stale learning removal is mandatory
- Retaining stale learning is prohibited

---

## 8. Separation from Research

Learning content:

### Is Downstream of Validated Research
- Learning content derives from validated research
- Research validation precedes learning creation
- Learning is downstream of research
- Learning without research validation is prohibited

### Must Not Ingest Raw Research Directly
- Learning does not ingest raw research
- Research must be processed before learning creation
- Direct research ingestion is prohibited
- Raw research in learning is a violation

---

## 9. Human Control

Human may:

### Pause Learning
- Human can pause learning at any time
- Pause is immediate and effective
- Pause authority is absolute
- Pause cannot be overridden

### Request Alternative Modality
- Human can request different modality
- Modality change is honored
- Modality request authority is absolute
- Modality requests cannot be denied

### Demand Justification
- Human can demand learning justification
- Justification must be provided
- Justification demand authority is absolute
- Justification cannot be refused

### Disable a Learning Stream
- Human can disable entire learning stream
- Disable is permanent until re-enabled
- Disable authority is absolute
- Disabled streams cannot auto-reactivate

---

## 10. Audit Trail

For each learning artefact record:

### Purpose
- Purpose of the learning artefact
- What the learning is intended to achieve
- Purpose is mandatory in audit trail
- Missing purpose in audit trail is a violation

### Modality
- Modality used for delivery
- Spatial, VR, visual, or text
- Modality is mandatory in audit trail
- Missing modality in audit trail is a violation

### Trigger
- What triggered the learning artefact
- Event or condition that caused learning creation
- Trigger is mandatory in audit trail
- Missing trigger in audit trail is a violation

### Feedback Type
- Type of feedback provided
- Outcome mirroring, trade-offs, or uncertainty
- Feedback type is mandatory in audit trail
- Missing feedback type in audit trail is a violation

### Observed Impact (If Any)
- Impact observed from learning
- Changes in judgment, clarity, or decisions
- Observed impact is recorded when available
- Impact recording is optional but recommended

### Audit Trail Format

```
[timestamp] [learning_id] [purpose] [modality] [trigger] [feedback_type] [impact]
```

Example:
```
2024-12-13T10:23:45Z learning-001 "improve_judgment" "spatial" "system_update" "outcome_mirroring" "clarity_increased"
```

### Audit Trail Retention

- All learning audit trails: Retained for 2 years minimum
- Impact tracking: Retained for learning effectiveness analysis
- Modality preferences: Retained for personalization
- Minimum retention: 2 years for all learning logs

## Change Control

This contract changes only via explicit revision:

### Explicit Revision
- Contract changes must be explicit and documented
- No ad-hoc edits to learning principles or experience rules
- Changes require version control
- Changes are traceable and auditable

### Documented Rationale
- Every contract change must include rationale:
  - What changed in the contract
  - Why it changed
  - What it affects
  - Expected impact on learning quality and experience

### Rollback Plan
- Every contract change must have rollback plan
- Previous contract versions must be preserved
- Rollback procedure must be documented
- Rollback can be executed at any time

---

## Enforcement

Violations of this contract include:
- Coercive learning tactics
- Gamification for addiction
- Shaming learners
- Optimizing for completion over understanding
- Urgency cues, streaks, or leaderboards
- Prescribing identity or labeling learners
- Using speed, volume, or completion as progress signals
- Missing or incomplete audit trails

Violations must be:
- Logged as incidents
- Reviewed by system owner
- Corrected immediately
- Documented in change logs

---

## Amendments

Changes to learning intent, experience principles, modality priority, interaction rules, or feedback discipline require:
- Impact assessment on learning quality and experience
- Testing with representative learners
- Approval from system owner
- Version increment
- Documentation in change logs
